## Goal

`CLAUDE.md` and `DEVELOPMENT.md` describe `symbolindex.json` / `codeindex.json` as committed
artifacts and instruct agents to use `lookup_symbol` / `get_impact` as mandatory pre-steps.
Neither file exists. This plan: (1) generates and commits the three artifact files so they exist,
and (2) softens four doc lines in `CLAUDE.md` and three in `DEVELOPMENT.md` to advisory wording
that accurately reflects the "factory-regenerated, may lag HEAD" model.

---

## Architecture

No new services, models, or routes. All changes are:
- File generation via the `codeindex` CLI (already installed at `/usr/local/bin/codeindex`)
- `git add` / `git commit` of the generated files
- In-place edits to two markdown documentation files

The committed artifacts (`codeindex.json`, `symbolindex.json`, `docs/codeindex-hotspots.md`)
will be regenerated by the factory's `regen-codeindex` workflow node on every subsequent
implementation run. This plan establishes the baseline — stale-but-present beats absent.

- `git` for the commit step

## Task 1 — Generate and commit codeindex baseline artifacts

**Files:** `codeindex.json`, `symbolindex.json`, `docs/codeindex-hotspots.md`

### Steps

1. **Pre-flight: confirm codeindex is available**

   ```bash
   which codeindex && codeindex --version
   ```

   Expected output: path + version string. If absent, abort — the generator is not available
   outside the factory container; do not proceed.

2. **Check no partial artifacts exist**

   ```bash
   ls -la codeindex.json symbolindex.json docs/codeindex-hotspots.md 2>/dev/null || true
   ```

   Expected: files absent (first run) or present with factory-generated content. Either state
   is safe; the CLI overwrites them.

3. **Generate the dependency graph**

   ```bash
   codeindex analyze .
   ```

   Expected: `codeindex.json` written to repo root with non-zero byte count.
   Verify: `ls -lh codeindex.json` — must be > 0 bytes.

4. **Generate the symbol index**

   ```bash
   codeindex symbols . --inline
   ```

   Expected: `symbolindex.json` written to repo root.
   Verify: `ls -lh symbolindex.json` — must be > 0 bytes.

5. **Generate the hotspot report**

   ```bash
   codeindex high-blast > docs/codeindex-hotspots.md
   ```

   Expected: `docs/codeindex-hotspots.md` overwritten with real hotspot data (file list with
   blast-radius scores). If no high-blast files are detected the file may contain only a header
   line — that is still valid (not an error).

6. **Verify the three artifacts are non-empty**

   ```bash
   for f in codeindex.json symbolindex.json docs/codeindex-hotspots.md; do
     bytes=$(wc -c < "$f")
     [ "$bytes" -gt 0 ] && echo "OK: $f ($bytes bytes)" || echo "FAIL: $f is empty"
   done
   ```

   Expected: three `OK:` lines.

7. **Stage and commit the artifacts**

   ```bash
   git add codeindex.json symbolindex.json docs/codeindex-hotspots.md
   git commit -m "chore: commit initial codeindex baseline (#200)"
   ```

   Expected: commit created with all three files. `git show --stat` should list the three files.

## Task 2 — Soften CLAUDE.md codeindex instructions to advisory

**Files:** `CLAUDE.md`

### Steps

1. **Verify the exact text to be replaced still exists**

   ```bash
   grep -n "lookup_symbol MCP tool before grepping" CLAUDE.md
   ```

   Expected: a hit on the line corresponding to the old mandatory wording. If absent, the text
   was already updated — skip to step 5.

2. **Apply the advisory wording**

   Replace lines 116-119 of `CLAUDE.md`. The spec's exact before/after:

   **Before (lines 116-119):**
   ```
   Symbol index `symbolindex.json` + dependency graph `codeindex.json` are maintained in-repo (committed artifacts).
   Use the `lookup_symbol` MCP tool before grepping for any function or class.
   Use `get_impact` before modifying a high-blast file (see `docs/codeindex-hotspots.md` for the current list).
   Local interactive visualization: see [DEVELOPMENT.md](DEVELOPMENT.md#codeindex-local-visualization).
   ```

   **After (replace with):**
   ```
   Symbol index `symbolindex.json` + dependency graph `codeindex.json` are committed artifacts,
   regenerated by the factory — they may lag HEAD. When the codeindex MCP server is available,
   prefer `lookup_symbol` over grepping for a function/class and `get_impact` before touching a
   high-blast file (see `docs/codeindex-hotspots.md`); otherwise fall back to grep/Read.
   Local interactive visualization: see [DEVELOPMENT.md](DEVELOPMENT.md#codeindex-local-visualization).
   ```

   Use the Edit tool (or `sed -i`) targeting the exact old string. The old string is unique in the file.

3. **Verify the mandatory wording is gone**

   ```bash
   grep -c "Use the \`lookup_symbol\` MCP tool before grepping" CLAUDE.md || true
   ```

   Expected: `0`.

4. **Verify the advisory wording is present**

   ```bash
   grep -c "prefer \`lookup_symbol\` over grepping" CLAUDE.md
   ```

   Expected: `1`.

5. **Stage and commit**

   ```bash
   git add CLAUDE.md
   git commit -m "docs(#200): soften codeindex CLAUDE.md instructions to advisory-with-fallback"
   ```

   Expected: commit created listing `CLAUDE.md`.
